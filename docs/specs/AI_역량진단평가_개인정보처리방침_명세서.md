## 1. 개인정보 수집 현황

### 1.1 수집 원칙
- **최소 수집 원칙**: 서비스 제공에 필요한 최소한의 개인정보만 수집
- **선택적 수집**: 진단 평가 자체는 개인정보 없이 시작 가능
- **명시적 동의**: 개인정보 수집 시점에 명시적 동의 요구

### 1.2 수집 단계별 요약

| 단계 | 수집 항목 | 필수/선택 | 동의 필요 여부 |
|------|----------|----------|---------------|
| 진단 시작 | 닉네임, IP 주소(해시), User-Agent, UTM 파라미터 | 선택/자동 | 불필요 (익명) |
| 진단 진행 | 답변 데이터 (익명) | 필수 | 불필요 (익명) |
| 결과 조회 | 없음 (세션 키 기반) | - | 불필요 |
| AI 리포트 생성 | 없음 | - | 불필요 |
| 공유 링크 생성 | 닉네임 (선택) | 선택 | 불필요 (공개 요약만) |
| PDF 다운로드 | 이메일, 이름, 회사명, 직급 | 필수/선택 | **필수** |
| 상담 문의 | 이메일, 이름, 회사명, 직급, 문의 내용 | 필수/선택 | **필수** |

## 2. 수집 시점별 개인정보 항목

### 2.1 진단 시작 시점 (POST /api/assessment/start)

#### 수집 항목
```typescript
{
  nickname?: string;           // 선택: 사용자 이름
  locale: 'ko' | 'en';         // 필수: 언어 설정
  utm_source?: string;         // 선택: 마케팅 소스
  utm_medium?: string;         // 선택: 마케팅 미디어
  utm_campaign?: string;       // 선택: 마케팅 캠페인
  device_type?: string;        // 선택: 기기 유형
  // 자동 수집
  ip_address: string;          // 자동: IP 주소 (해시 처리)
  user_agent: string;          // 자동: 브라우저 정보
}
```

#### 개인정보 보호 조치
- **닉네임**: 선택 항목, 진단 결과와 연결되지만 개인 식별 불가
- **IP 주소**: SHA-256 해시 처리 (16자리), 원본 저장 안 함
- **UTM 파라미터**: 마케팅 분석용, 개인 식별 불가

#### 저장 위치
- 테이블: `uslab_assessment_sessions`
- 필드: `nickname`, `ip_address` (해시), `user_agent`, `utm_*`

---

### 2.2 진단 답변 시점 (POST /api/assessment/answer)

#### 수집 항목
```typescript
{
  answers: {
    'P1-1': 1-5,  // Part 1 문항 답변
    'P1-2': 1-5,
    'P1-3': 1-5,
    'P1-4': 1-5,
    'P2-1': 1-5,  // Part 2 문항 답변
    'P2-2': 1-5,
    'P2-3': 1-5,
    'P2-4': 1-5,
    'P3-1': 1-5,  // Part 3 문항 답변
    'P3-2': 1-5,
    'P3-3': 1-5,
    'P3-4': 1-5,
  }
}
```

#### 개인정보 보호 조치
- **익명 데이터**: 답변만 저장, 개인 식별 정보 없음
- **세션 키 기반**: `session_key`로만 연결, 개인 식별 불가
- **RLS 정책**: 서버 전용 접근, 클라이언트 직접 접근 불가

#### 저장 위치
- 테이블: `uslab_assessment_sessions`
- 필드: `answers` (JSONB)

---

### 2.3 PDF 다운로드 시점 (POST /api/assessment/download)

#### 수집 항목
```typescript
{
  email: string;                // 필수: 이메일 주소
  full_name?: string;           // 선택: 사용자 이름
  company?: string;             // 선택: 회사명
  job_title?: string;          // 선택: 직급
  locale: 'ko' | 'en';         // 필수: 언어 설정
  consent: {
    required: boolean;          // 필수: 동의 여부
    version: string;           // 필수: 동의 버전
  },
  meta?: {
    utm?: Record<string, string>;
    referrer?: string;
    user_agent?: string;
    ip_hash?: string;          // 자동: IP 해시
  }
}
```

#### 개인정보 보호 조치
- **명시적 동의**: 체크박스 필수 동의
- **동의 버전 관리**: `consent.version`으로 버전 추적
- **IP 해시 처리**: SHA-256 해시 (16자리)
- **이메일 정규화**: 소문자 변환 및 공백 제거

#### 저장 위치
- 테이블: `uslab_assessment_download_leads`
- 필드: `email`, `full_name`, `company`, `job_title`, `consent_version`, `meta`

#### 동의 고지 문구
```
결과 리포트 발송을 위한 개인정보(이메일, 이름, 회사명, 직급) 수집·이용 동의 
(보유기간: 회원 탈퇴 시까지)
[개인정보처리방침 보기: /ko/privacy]
```

## 3. 개인정보 처리 목적

### 3.1 진단 평가 서비스 제공
- **목적**: AI 역량 진단 평가 및 결과 리포트 제공
- **수집 항목**: 닉네임(선택), 답변 데이터(익명), IP 주소(해시)
- **법적 근거**: 서비스 제공을 위한 최소한의 정보 수집

### 3.2 결과 리포트 발송
- **목적**: PDF 리포트 이메일 발송 및 다운로드 링크 제공
- **수집 항목**: 이메일(필수), 이름, 회사명, 직급(선택)
- **법적 근거**: 사용자 명시적 동의

### 3.3 상담 문의 응대
- **목적**: 사용자 문의 및 상담 요청 처리
- **수집 항목**: 이메일(필수), 이름, 회사명, 직급, 문의 내용(선택)
- **법적 근거**: 사용자 명시적 동의

### 3.4 서비스 개선 및 분석
- **목적**: 서비스 품질 개선, 통계 분석, 벤치마크 지표 고도화
- **수집 항목**: 익명화된 진단 데이터, 접속 로그(IP 해시)
- **법적 근거**: 서비스 개선을 위한 통계 목적

### 3.5 마케팅 분석
- **목적**: 마케팅 캠페인 효과 분석
- **수집 항목**: UTM 파라미터, 접속 경로
- **법적 근거**: 개인 식별 불가능한 통계 데이터

## 5. 개인정보 보호 조치

### 5.1 기술적 보호조치

#### 암호화
- **전송 구간**: HTTPS (TLS 1.2 이상)
- **저장 데이터**: 데이터베이스 암호화 (Supabase 기본 제공)
- **IP 주소**: SHA-256 해시 처리 (16자리)
- **토큰**: SHA-256 해시만 저장 (원본 토큰 DB 저장 안 함)

#### 접근 제어
- **RLS (Row Level Security)**: 테이블별 접근 권한 제어
  - `uslab_assessment_sessions`: 서버 전용
  - `uslab_assessment_download_leads`: 서버 전용
  - `uslab_assessment_shares`: SELECT만 anon 허용 (revoked_at IS NULL)
- **세션 키**: 랜덤 생성, URL 파라미터로 전달 (쿠키 미사용)

#### 데이터 최소화
- **공유 링크**: PII 제거된 `public_summary`만 저장
- **익명 데이터**: 진단 답변은 개인 식별 불가능한 형태로 저장
- **선택적 수집**: 진단 평가 자체는 개인정보 없이 시작 가능

---

### 5.2 관리적 보호조치

#### 접근 권한 관리
- **최소 권한 원칙**: 서비스 제공에 필요한 최소한의 권한만 부여
- **접근 로그**: 관리자 접근 이력 기록 (구현 권장)

#### 교육 및 인식
- **개인정보 보호 교육**: 개인정보 처리 담당자 대상 정기 교육
- **보안 인식**: 개인정보 보호 중요성 인식 제고

#### 내부 관리계획
- **개인정보 보호 관리체계**: ISO/IEC 27001:2022 인증 기반 운영
- **정기 점검**: 개인정보 보호 정책 및 절차 정기 점검

---

### 5.3 물리적 보호조치

#### 서버 보안
- **클라우드 인프라**: Supabase (AWS 기반) 보안 인증 기반
- **데이터 센터**: 물리적 접근 제어, 화재/재해 대비
