## 1. 개인정보 수집 현황

### 1.1 수집 원칙
- **최소 수집 원칙**: 서비스 제공에 필요한 최소한의 개인정보만 수집
- **선택적 수집**: 게스트 입장 가능한 웨비나는 개인정보 없이 참여 가능
- **명시적 동의**: 개인정보 수집 시점에 명시적 동의 요구

### 1.2 수집 단계별 요약

| 단계 | 수집 항목 | 필수/선택 | 동의 필요 여부 |
|------|----------|----------|---------------|
| 웨비나 등록 | 이름, 이메일, 회사명, 직급, 연락처 | 필수/선택 | **필수** |
| 웨비나 참여 | 이름, 이메일 (등록 정보) | 필수 | 등록 시 동의 |
| 채팅 참여 | 이름, 프로필 정보 | 자동 | 등록 시 동의 |
| Q&A 참여 | 이름, 질문 내용 | 필수 | 등록 시 동의 |
| 설문 응답 | 이름, 응답 데이터 | 필수 | 등록 시 동의 |
| 관리자 계정 | 이메일, 이름, 조직 정보 | 필수 | **필수** |
| 회원가입 | 이메일, 이름, 비밀번호 | 필수 | **필수** |

## 2. 수집 시점별 개인정보 항목

### 2.1 웨비나 등록 시점 (POST /api/webinars/[webinarId]/register)

#### 수집 항목
```typescript
{
  email: string;                // 필수: 이메일 주소
  full_name: string;           // 필수: 사용자 이름
  company?: string;            // 선택: 회사명
  job_title?: string;          // 선택: 직급
  phone?: string;              // 선택: 연락처
  consent: {
    required: boolean;          // 필수: 동의 여부
    version: string;           // 필수: 동의 버전
  }
}
```

#### 개인정보 보호 조치
- **명시적 동의**: 체크박스 필수 동의
- **동의 버전 관리**: `consent.version`으로 버전 추적
- **이메일 정규화**: 소문자 변환 및 공백 제거
- **게스트 입장**: `access_policy === 'guest_allowed'`인 경우 등록 없이 참여 가능

#### 저장 위치
- 테이블: `webinar_registrants`
- 필드: `email`, `full_name`, `company`, `job_title`, `phone`, `consent_version`

#### 동의 고지 문구
```
웨비나 참여를 위한 개인정보(이메일, 이름, 회사명, 직급, 연락처) 수집·이용 동의 
(보유기간: 웨비나 종료 후 1년 또는 회원 탈퇴 시까지)
[개인정보처리방침 보기: /privacy]
```

---

### 2.2 회원가입 시점 (POST /api/auth/signup)

#### 수집 항목
```typescript
{
  email: string;                // 필수: 이메일 주소
  password: string;             // 필수: 비밀번호 (해시 처리)
  full_name?: string;           // 선택: 사용자 이름
  organization_type: 'agency' | 'client'; // 필수: 조직 유형
  organization_name?: string;   // 선택: 조직명
  consent: {
    required: boolean;          // 필수: 동의 여부
    version: string;           // 필수: 동의 버전
  }
}
```

#### 개인정보 보호 조치
- **명시적 동의**: 체크박스 필수 동의
- **비밀번호 해시**: bcrypt 등 단방향 해시 처리, 원본 저장 안 함
- **이메일 인증**: 회원가입 후 이메일 인증 필수

#### 저장 위치
- 테이블: `users` (Supabase Auth), `profiles`
- 필드: `email`, `full_name`, `organization_type`, `organization_name`, `consent_version`

#### 동의 고지 문구
```
회원가입을 위한 개인정보(이메일, 이름, 조직 정보) 수집·이용 동의 
(보유기간: 회원 탈퇴 시까지)
[개인정보처리방침 보기: /privacy]
```

---

### 2.3 웨비나 참여 시점 (실시간 인터랙션)

#### 수집 항목
```typescript
{
  // 채팅 메시지
  message: string;              // 필수: 채팅 내용
  user_id: string;             // 자동: 사용자 ID
  display_name: string;        // 자동: 표시 이름
  
  // Q&A 질문
  question: string;             // 필수: 질문 내용
  user_id: string;             // 자동: 사용자 ID
  display_name: string;        // 자동: 표시 이름
  
  // 설문 응답
  form_id: string;             // 필수: 설문 ID
  responses: Record<string, any>; // 필수: 응답 데이터
  user_id: string;             // 자동: 사용자 ID
}
```

#### 개인정보 보호 조치
- **등록 정보 기반**: 웨비나 등록 시 동의한 정보만 사용
- **익명 옵션**: 일부 웨비나에서 익명 채팅/Q&A 가능
- **모더레이션**: 관리자가 부적절한 내용 삭제 가능

#### 저장 위치
- 테이블: `messages`, `questions`, `form_responses`
- 필드: `user_id`, `display_name`, `message/question/responses`

---

### 2.4 웨비나 통계 수집 시점

#### 수집 항목
```typescript
{
  // 참여 통계
  webinar_id: string;          // 자동: 웨비나 ID
  user_id: string;             // 자동: 사용자 ID
  join_time: timestamp;       // 자동: 참여 시작 시간
  leave_time?: timestamp;      // 자동: 참여 종료 시간
  watch_duration: number;      // 자동: 시청 시간 (초)
  
  // 행동 데이터
  interactions: {
    chat_messages: number;     // 채팅 메시지 수
    questions_asked: number;    // 질문 수
    forms_submitted: number;   // 설문 제출 수
  }
}
```

#### 개인정보 보호 조치
- **통계 목적**: 서비스 개선 및 분석 목적
- **익명화**: 개인 식별 불가능한 형태로 가공하여 분석
- **RLS 정책**: 관리자만 접근 가능

#### 저장 위치
- 테이블: `webinar_analytics`, `user_webinar_stats`
- 필드: 통계 데이터 (개인 식별 최소화)

## 3. 개인정보 처리 목적

### 3.1 웨비나 서비스 제공
- **목적**: 웨비나 등록, 참여, 실시간 인터랙션 제공
- **수집 항목**: 이름, 이메일, 회사명, 직급, 연락처
- **법적 근거**: 서비스 제공을 위한 최소한의 정보 수집 및 사용자 명시적 동의

### 3.2 회원 관리 및 인증
- **목적**: 계정 생성, 로그인, 권한 관리
- **수집 항목**: 이메일, 비밀번호(해시), 이름, 조직 정보
- **법적 근거**: 서비스 제공을 위한 필수 정보 및 사용자 명시적 동의

### 3.3 실시간 인터랙션 제공
- **목적**: 채팅, Q&A, 설문조사 등 실시간 상호작용 기능 제공
- **수집 항목**: 이름, 메시지/질문/응답 내용
- **법적 근거**: 웨비나 등록 시 동의한 범위 내 처리

### 3.4 웨비나 운영 및 관리
- **목적**: 웨비나 진행, 참여자 관리, 콘텐츠 모더레이션
- **수집 항목**: 등록 정보, 참여 기록, 인터랙션 데이터
- **법적 근거**: 서비스 제공을 위한 필수 정보

### 3.5 통계 분석 및 서비스 개선
- **목적**: 웨비나 참여 통계, 사용자 행동 분석, 서비스 품질 개선
- **수집 항목**: 익명화된 참여 데이터, 통계 정보
- **법적 근거**: 서비스 개선을 위한 통계 목적

### 3.6 고객 지원 및 문의 응대
- **목적**: 사용자 문의 및 기술 지원
- **수집 항목**: 이메일, 이름, 문의 내용
- **법적 근거**: 사용자 명시적 동의

## 5. 개인정보 보호 조치

### 5.1 기술적 보호조치

#### 암호화
- **전송 구간**: HTTPS (TLS 1.2 이상)
- **저장 데이터**: 데이터베이스 암호화 (Supabase 기본 제공)
- **비밀번호**: bcrypt 등 단방향 해시 처리, 원본 저장 안 함
- **민감 정보**: 이메일, 개인 식별 정보 암호화 저장

#### 접근 제어
- **RLS (Row Level Security)**: 테이블별 접근 권한 제어
  - `webinar_registrants`: 등록자 본인 및 웨비나 관리자만 접근
  - `messages`: 웨비나 참여자 및 관리자만 접근
  - `questions`: 웨비나 참여자 및 관리자만 접근
  - `form_responses`: 응답자 본인 및 웨비나 관리자만 접근
- **JWT 토큰**: 인증 토큰 기반 접근 제어
- **역할 기반 접근**: Super Admin, Agency, Client 등 역할별 권한 분리

#### 데이터 최소화
- **선택적 수집**: 게스트 입장 가능한 웨비나는 개인정보 없이 참여 가능
- **필수 항목 최소화**: 서비스 제공에 필요한 최소한의 정보만 수집
- **익명 옵션**: 일부 기능에서 익명 참여 가능

---

### 5.2 관리적 보호조치

#### 접근 권한 관리
- **최소 권한 원칙**: 서비스 제공에 필요한 최소한의 권한만 부여
- **접근 로그**: 관리자 접근 이력 기록 (구현 권장)
- **정기 권한 검토**: 사용자 권한 정기 점검 및 조정

#### 교육 및 인식
- **개인정보 보호 교육**: 개인정보 처리 담당자 대상 정기 교육
- **보안 인식**: 개인정보 보호 중요성 인식 제고
- **보안 정책**: 내부 보안 정책 수립 및 준수

#### 내부 관리계획
- **개인정보 보호 관리체계**: ISO/IEC 27001:2022 인증 기반 운영
- **정기 점검**: 개인정보 보호 정책 및 절차 정기 점검
- **사고 대응**: 개인정보 유출 시 신속한 대응 절차 수립

---

### 5.3 물리적 보호조치

#### 서버 보안
- **클라우드 인프라**: Supabase (AWS 기반) 보안 인증 기반
- **데이터 센터**: 물리적 접근 제어, 화재/재해 대비
- **백업 및 복구**: 정기적인 데이터 백업 및 복구 절차

- 현재 제3자 제공 없음

### 6.2 개인정보 처리 위탁

#### 위탁 현황

| 위탁 업체 | 위탁 업무 | 위탁 항목 | 보유기간 |
