좋아. 지금 너가 정리한 **“uslab.ai Data-First Admin 스타일”**을 inev.ai 리빌드에 적용하려면, **디자인 원칙을 먼저 고정하고 → 화면별로 ‘교체 규칙’을 적용**하면 돼. 아래는 Cursor Agent가 그대로 따라가며 리팩토링할 수 있는 **UI Implementation Plan(요약 체크리스트)**야.

---

## 1) 목표 UI 원칙 7개

1. **화이트 카드 + 옅은 보더**(색 배경/그라데이션 금지)
2. **컬러는 액션/상태/차트 라인에만**(데이터 카드 배경에 컬러 금지)
3. **라벨은 작고 옅게 / 값은 크고 진하게**(숫자가 주인공)
4. **상태는 배지/닷으로만 표현**(Live/Closed 등)
5. **테이블은 헤더 연한 bg + row bottom border만**
6. **차트는 라인 중심(채움 최소)** + 격자는 아주 약하게
7. **대시보드는 “정보 위계”가 먼저**: KPI → 추이 → 상세(리스트/테이블)

---

## 2) 공통 디자인 토큰(반드시 고정)

* 배경: 페이지는 아주 연한 회색, 카드는 흰색
* 보더: Gray-200 수준, 1px
* 텍스트: Gray-900 / 라벨 Gray-500 / 보조 Gray-400
* 강조색(Primary): 버튼/아이콘/차트 라인만
* 라운드: `rounded-lg` 정도(알약형 pill 남발 금지)
* 그림자: 최소(또는 없음), 필요 시 아주 약하게

> 핵심: “예쁘게”가 아니라 **“관리자용 신뢰감/가독성”**이 목적.

---

## 3) 컴포넌트 교체 규칙(기존 EventFlow UI를 이렇게 바꾼다)

### 3.1 KPI 카드(상단 숫자 카드)

* ✅ 카드 배경: **흰색**
* ✅ 구분: **border만**
* ✅ 구조 고정:

  * Label (작고 흐리게)
  * Value (크고 진하게)
  * Sub-info (작게)
* ✅ 아이콘: 배경 없이 아이콘만 색(또는 아주 옅은 원형 배경 1개)
* ✅ Live 상태: 카드색이 아니라 **Dot + Badge**

### 3.2 그래프(추이)

* ✅ 라인 두께 2~3px
* ✅ fill은 없거나 아주 옅게(opacity 0.1)
* ✅ grid는 흐리게, 세로선 점선 권장
* ✅ 범례는 하단/상단 한 곳으로 고정
* ❌ 그라데이션 과다 / 굵은 선 / 배경색 영역 과다 금지

### 3.3 테이블/리스트

* ✅ 헤더: `bg-gray-50` 또는 흰색 + border-bottom
* ✅ 행: bottom border만
* ✅ 액션: 텍스트 링크 or outline 버튼(“대시보드→” 같은 건 링크 느낌)
* ❌ 진한 컬러 헤더, 과한 pill 버튼 금지

### 3.4 네비게이션(GNB/사이드)

* ✅ 흰 배경 + 하단 보더
* ✅ 아이콘은 line icon 통일
* ✅ 현재 메뉴만 강조색(배경색 강조 최소)
* ❌ 그라데이션/큰 배경색 블록 금지

---

## 4) inev.ai 화면별 적용 우선순위(리빌드 순서랑 맞춰서)

리빌드 진행 흐름에 맞춰 UI도 같이 “바뀐 영역부터” 통일하면 부담이 적어.

1. **Admin: Client/Event 목록 화면**

* 카드/테이블/버튼 스타일 표준화부터

2. **Event 상세(설정/모듈 ON-OFF)**

* 설정 패널 디자인(좌: 섹션, 우: 콘텐츠)

3. **등록자/설문 응답 리스트**

* 테이블 표준화(검색/필터/다운로드 버튼 위치 고정)

4. **이메일 캠페인 화면**

* Novel 에디터 영역 포함: “편집 패널 + 미리보기 패널” 두 칼럼을 카드로 정리

5. **웨비나 대시보드(가장 중요)**

* KPI → 차트 → 참여자 테이블 순서로 재배치
* 실시간 요소는 “작은 배지/작은 숫자”로만 강조(과한 애니메이션 금지)

---

## 5) 성공 조건(DoD)

* 어떤 화면을 봐도 “같은 제품”처럼 보임(카드/테이블/버튼 일관성)
* KPI는 3초 훑으면 핵심 숫자가 읽힘
* 웨비나 대시보드에서 **색이 ‘정보’가 아니라 ‘강조’로만** 쓰임
* 관리자 화면에서 과한 리렌더/폴링이 UI 때문에 더 늘지 않음(성능 영향 최소)

---

## 6) Cursor Agent에게 전달할 “실행 지시” 한 줄

> “EventFlow UI의 컬러 배경/그라데이션을 모두 제거하고, White Card + Light Border 기반의 Data-First Admin 스타일로 공통 컴포넌트(카드/버튼/테이블/배지/차트)를 먼저 만든 뒤, Admin→Event→웨비나 대시보드 순으로 적용한다.”

---

원하면, 다음 메시지에서 **대시보드 와이어(정보 배치)**를 텍스트로 딱 그려줄게:

* KPI 6개를 2줄로 배치하는 규칙(너가 말한 ‘1~2줄로 깔끔하게’)
* 웨비나용(실시간) vs 설문/등록용(비실시간) 대시보드 레이아웃 차이
  이렇게 “화면 구조”까지 확정해주면 Cursor가 UI 리빌드 속도가 훨씬 빨라져.
