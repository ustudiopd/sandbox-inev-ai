## 📘 문서에 넣을 최종 정책 문장 (확정본 · 복붙)

```md
### 클라이언트 도메인 및 Vercel 프로젝트 정책 (확정)

WERT 클라이언트의 Vercel 프로젝트명은 `wert-inev-ai`로 고정한다.

고객이 커스텀 도메인을 요청할 경우, 해당 도메인을
클라이언트의 **유일한 대표 도메인(canonical)** 으로 사용한다.
예: `https://ai-native-research.com`

- 모든 외부 발송 링크(이메일, 문자, UTM, QR, Short Link)는
  대표 도메인 기준으로만 생성한다:
  `https://{canonical}/s/{code}?...`

- `{client}.inev.ai` 서브도메인은 생성하지 않는다.

- 내부 운영, QA, 장애 대응을 위해
  Vercel Preview 도메인(`https://wert-inev-ai.vercel.app`)을 사용하며,
  해당 URL은 고객에게 공개하지 않는다.
```

이 블록을:

* `inev_리빌딩_전체구현계획.md`
* `inev.ai_개발_배포_운영_명세서.md`
* `inev_리빌딩_매직링크.md`

에 **동일하게** 넣으면 정책 봉인 완료야.
