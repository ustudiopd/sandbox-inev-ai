## UTM이란?

UTM (Urchin Tracking Module)은 마케팅 캠페인의 성과를 추적하기 위한 URL 파라미터입니다.

### 주요 UTM 파라미터

- **utm_source**: 유입 경로 (예: newsletter, google, facebook)
- **utm_medium**: 매체 유형 (예: email, cpc, sms)
- **utm_campaign**: 캠페인 이름 (예: january_2026, product_launch)
- **utm_term**: 키워드 (검색 광고용, 선택사항)
- **utm_content**: 콘텐츠 구분 (A/B 테스트용, 선택사항)

### 예시

```
https://eventflow.kr/event/149403?utm_source=newsletter&utm_medium=email&utm_campaign=january_2026
```

이 링크를 클릭한 사용자는:
- **Source**: newsletter (뉴스레터)
- **Medium**: email (이메일)
- **Campaign**: january_2026 (1월 캠페인)

으로 분류되어 집계됩니다.

## 시스템 개요

### 제공 기능

1. **캠페인 링크 생성**: 템플릿 선택만으로 UTM 링크 자동 생성
2. **Visit 추적**: 링크 클릭 시 유입 로그 자동 수집
3. **전환 추적**: 등록/제출 시 UTM 파라미터 자동 저장
4. **성과 분석**: Visits, Conversions, CVR 집계 및 대시보드 제공

### 전체 흐름

```
1. 운영자가 링크 생성
   ↓
2. 생성된 링크를 마케팅 채널에 배포
   ↓
3. 사용자가 링크 클릭 → Visit 로그 수집
   ↓
4. 사용자가 등록/제출 → 전환 데이터 저장
   ↓
5. 대시보드에서 성과 확인
```

## 캠페인 링크 생성하기

### 1단계: 대시보드 접속

1. 클라이언트 대시보드 접속
2. 좌측 메뉴에서 "📈 광고/캠페인" 클릭
3. "캠페인 링크" 탭 선택

### 2단계: 새 링크 생성

1. "새 링크 생성" 버튼 클릭
2. **템플릿 선택** (필수)
   - 📧 뉴스레터: 이메일 발송용
   - 📱 문자 / 카카오: SMS/카카오톡 발송용
   - 🔍 구글 광고: Google Ads용
   - 📘 메타 광고: Facebook/Instagram Ads용
   - 🤝 파트너 / 제휴: 제휴사 링크용
   - 🧪 기타(커스텀): 직접 입력

3. **링크 이름 입력** (필수)
   - 예: "26년 1월 뉴스레터", "구글 광고 1차"
   - `utm_campaign`은 자동으로 생성됩니다 (slug 변환)

4. **전환 타겟 선택** (필수)
   - 추적하고 싶은 이벤트/설문 캠페인 선택

5. **랜딩 위치 선택** (필수)
   - `welcome`: 환영 페이지
   - `register`: 등록 페이지
   - `survey`: 설문 페이지

6. **시작일 입력** (선택사항)
   - 광고 시작일 설정 (해당 날짜부터 링크 활성화)

7. **고급 옵션** (선택사항)
   - UTM 파라미터 직접 수정 가능
   - 템플릿 기본값에서 변경 필요 시 사용

8. **생성** 버튼 클릭

### 3단계: 생성된 링크 확인

생성 완료 후 두 가지 URL이 제공됩니다:

1. **공유용 URL** (짧음, CID만)
   ```
   https://eventflow.kr/event/149403?cid=X5L2G9KV
   ```
   - 문자/카카오톡에 적합
   - 짧아서 공유하기 편함

2. **광고용 URL** (UTM 포함)
   ```
   https://eventflow.kr/event/149403?cid=X5L2G9KV&utm_source=newsletter&utm_medium=email&utm_campaign=...
   ```
   - 뉴스레터/이메일/광고 플랫폼에 적합
   - UTM 파라미터 포함으로 상세 추적 가능

**추천 링크**가 파란색 테두리로 강조 표시됩니다.

## 링크 배포하기

### 뉴스레터/이메일

**사용 링크**: 광고용 URL (UTM 포함)

**배포 방법**:
1. 생성된 광고용 URL 복사
2. 이메일 본문에 링크 삽입
3. 발송

**예시**:
```
안녕하세요,

이번 달 웨비나에 참여해주세요!
👉 https://eventflow.kr/event/149403?cid=X5L2G9KV&utm_source=newsletter&utm_medium=email&utm_campaign=january_2026
```

---

### 문자/카카오톡

**사용 링크**: 공유용 URL (CID만, 짧음)

**배포 방법**:
1. 생성된 공유용 URL 복사
2. 문자 메시지 또는 카카오톡에 링크 삽입
3. 발송

**예시**:
```
[이벤트라이브] 웨비나 참여 링크
https://eventflow.kr/event/149403?cid=X5L2G9KV
```

---

### 구글/메타 광고

**사용 링크**: 광고용 URL (UTM 포함)

**배포 방법**:
1. 생성된 광고용 URL 복사
2. 광고 플랫폼의 "최종 URL" 또는 "랜딩 페이지 URL"에 입력
3. 광고 게시

**주의사항**:
- UTM 파라미터가 이미 포함되어 있으므로, 광고 플랫폼의 자동 UTM 추가 기능은 비활성화하세요
- 광고 플랫폼에서 추가 UTM을 붙이면 중복되어 집계가 깨질 수 있습니다

---

### 제휴사/파트너

**사용 링크**: 광고용 URL (UTM 포함)

**배포 방법**:
1. 생성된 광고용 URL을 제휴사에 제공
2. 제휴사가 자신의 채널에 링크 배포

## 성과 확인하기

### 대시보드 접속

1. 클라이언트 대시보드 접속
2. 좌측 메뉴에서 "📈 광고/캠페인" 클릭
3. "전환 성과" 탭 선택 (기본)

### 날짜 범위 설정

1. 시작일 선택
2. 종료일 선택
3. "조회" 버튼 클릭

### 확인 가능한 데이터

#### 1. KPI 카드

- **전체 Visits**: 선택한 기간 동안의 총 유입 수 (세션별 첫 방문만 집계)
- **전체 전환 수**: 등록/제출 완료 수
- **전환율 (CVR)**: Visits 대비 Conversions 비율 (%)

#### 2. Source별 집계

- utm_source별 Visits, Conversions, CVR
- 예: newsletter (200 Visits, 50 Conversions, 25% CVR)

#### 3. Medium별 집계

- utm_medium별 Visits, Conversions, CVR
- 예: email (200 Visits, 50 Conversions, 25% CVR)

#### 4. Campaign별 집계

- utm_campaign별 Visits, Conversions, CVR
- 예: january_2026 (200 Visits, 50 Conversions, 25% CVR)

#### 5. 링크별 집계

- 생성된 링크별 Visits, Conversions, CVR
- 예: "26년 1월 뉴스레터" (200 Visits, 50 Conversions, 25% CVR)

### 데이터 해석

#### CVR (전환율)이란?

**CVR = (전환 수 / Visits 수) × 100**

예시:
- Visits: 100명
- Conversions: 25명
- CVR: 25%

**의미**: 100명이 방문했고, 그 중 25명이 등록/제출했다는 뜻입니다.

#### 좋은 CVR은?

- **이메일/뉴스레터**: 20-30% 이상
- **검색 광고**: 5-15%
- **소셜 미디어**: 2-10%
- **문자/카카오톡**: 10-25%

(업종/제품에 따라 다를 수 있습니다)

## 자주 묻는 질문

### Q1. 같은 링크를 여러 번 사용해도 되나요?

**A**: 네, 가능합니다. 같은 링크를 여러 채널에 배포해도 각 채널별로 집계됩니다. 단, UTM 파라미터가 동일하면 같은 캠페인으로 집계됩니다.

**예시**:
- 뉴스레터 A와 뉴스레터 B에서 같은 링크 사용 → 같은 Campaign으로 집계
- 뉴스레터와 구글 광고에서 다른 링크 사용 → 각각 다른 Campaign으로 집계

---

### Q2. 링크를 수정하면 기존 데이터는 어떻게 되나요?

**A**: 링크를 수정해도 기존에 저장된 전환 데이터는 변경되지 않습니다. 수정된 링크는 이후부터 새로운 UTM 파라미터로 집계됩니다.

**권장사항**: 링크를 수정하기보다는 새 링크를 생성하는 것을 권장합니다.

---

### Q3. Visit 수와 전환 수가 맞지 않아요

**A**: 다음을 확인해보세요:

1. **세션 만료**: Visit은 세션 기반입니다. 세션이 만료되면(30분) 새로운 Visit으로 집계됩니다.
2. **전환 시점**: Visit은 페이지 접속 시, 전환은 등록/제출 시 기록됩니다. 시간 차이가 있을 수 있습니다.
3. **직접 접속**: UTM 없이 직접 접속한 경우 Visit은 기록되지만, UTM이 없어서 "Direct (UTM 없음)"으로 분류됩니다.

---

### Q4. "Direct (UTM 없음)"이 많이 나와요

**A**: 다음을 확인해보세요:

1. **링크 배포 확인**: 생성된 링크가 제대로 배포되었는지 확인
2. **링크 클릭 확인**: 사용자가 실제로 링크를 클릭했는지 확인
3. **직접 접속**: 북마크나 직접 URL 입력으로 접속한 경우 UTM이 없습니다 (정상)

---

### Q5. CVR이 0%로 나와요

**A**: 다음을 확인해보세요:

1. **Visit 수집 확인**: Visit API가 정상 작동하는지 확인
2. **전환 데이터 확인**: 실제로 등록/제출이 발생했는지 확인
3. **날짜 범위 확인**: Visit과 전환이 같은 기간에 발생했는지 확인

---

### Q6. 링크를 삭제해도 되나요?

**A**: 링크를 삭제(archived)해도 기존 전환 데이터는 유지됩니다. 다만, 삭제된 링크는 더 이상 사용할 수 없습니다.

**권장사항**: 링크를 삭제하기보다는 "일시정지(paused)" 상태로 변경하는 것을 권장합니다.

---

### Q7. 여러 캠페인을 동시에 추적할 수 있나요?

**A**: 네, 가능합니다. 각 캠페인마다 별도의 링크를 생성하면 됩니다.

**예시**:
- "26년 1월 뉴스레터" 링크 → january_newsletter 캠페인
- "26년 2월 뉴스레터" 링크 → february_newsletter 캠페인

각각 독립적으로 집계됩니다.

---

### Q8. A/B 테스트를 하고 싶어요

**A**: `utm_content` 파라미터를 활용하세요.

**방법**:
1. 같은 캠페인으로 두 개의 링크 생성
2. 하나는 `utm_content=version_a`, 다른 하나는 `utm_content=version_b`로 설정
3. 각각 다른 채널에 배포
4. 대시보드에서 `utm_content`별 집계 확인
