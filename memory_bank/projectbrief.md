# 프로젝트 개요 (Project Brief)

## 서비스 정보
- **서비스 이름**: Inev.ai
- **도메인**: inev.ai
- **설명**: B2B2C 멀티테넌시 웨비나 플랫폼

## 1. 프로젝트 목표  

### 핵심 문제 해결
- **B2B2C 멀티테넌시 웨비나 플랫폼 구축**: 에이전시가 여러 클라이언트를 관리하고, 각 클라이언트가 독립적으로 웨비나를 운영할 수 있는 SaaS 플랫폼 제공
- **데이터 격리 및 보안**: 계층형 멀티테넌시 구조에서 각 조직의 데이터를 완전히 격리하여 보안성 확보
- **인터랙티브 웨비나 경험**: YouTube 생중계 기반으로 실시간 채팅, Q&A, 퀴즈, 추첨 등 상호작용 기능 제공

### 최종 목표 및 기대 효과
- **확장 가능한 SaaS 플랫폼**: 에이전시 단위로 여러 클라이언트를 온보딩하여 비즈니스 확장
- **화이트레이블 솔루션**: 각 클라이언트가 자체 브랜딩으로 웨비나 서비스 제공
- **운영 효율성**: 통합 대시보드와 리포트로 웨비나 운영 및 성과 분석
- **참여자 경험 향상**: 실시간 상호작용으로 웨비나 참여도 및 만족도 향상

- ✅ 웨비나 시청 (YouTube 임베드)
- ✅ 실시간 채팅
- ✅ Q&A 질문 등록
- ✅ 참여자 목록 확인
- ✅ 전체화면 모드

## 3. 타겟 사용자  

### 슈퍼 관리자
- **역할**: 플랫폼 전체 관리자
- **목적**: 에이전시 관리, 플랫폼 전반 모니터링
- **주요 활동**: 에이전시 생성, 통계 확인, 감사 로그 확인

### 에이전시
- **역할**: 여러 클라이언트를 관리하는 중간 조직
- **목적**: 클라이언트 온보딩 및 운영 지원
- **주요 활동**: 클라이언트 생성/관리, 리포트 확인, 도메인 관리

### 클라이언트 (행사 운영자)
- **역할**: 실제 웨비나를 주최하는 기업/조직
- **목적**: 웨비나 생성 및 운영
- **주요 활동**: 웨비나 생성, 브랜딩 설정, 운영 콘솔에서 상호작용 관리

### 참여자
- **역할**: 웨비나를 시청하는 최종 사용자
- **목적**: 웨비나 시청 및 상호작용 참여
- **주요 활동**: 웨비나 시청, 채팅, 질문 등록, 퀴즈/추첨 참여

## 4. 주요 특징

### 멀티테넌시
- **계층형 구조**: Super Admin → Agency → Client → Participant
- **데이터 격리**: RLS 정책으로 완전한 데이터 격리
- **독립적 운영**: 각 클라이언트가 독립적으로 웨비나 운영

### 실시간 기능
- **실시간 채팅**: Supabase Realtime으로 즉시 메시지 전파
- **실시간 Q&A**: 질문 및 답변 실시간 업데이트
- **Presence**: 참여자 수 및 목록 실시간 표시

### 확장성
- **서버리스 아키텍처**: Vercel Serverless Functions로 자동 스케일링
- **모듈화된 컴포넌트**: 재사용 가능한 컴포넌트 구조
- **API 중심 설계**: 프론트엔드와 백엔드 분리

### 보안
- **RLS 전면 적용**: 데이터베이스 레벨 보안
- **권한 기반 접근 제어**: 역할별 세분화된 권한
- **감사 로그**: 모든 중요한 작업 기록

## 5. 기술적 특징

### 프론트엔드
- **Next.js App Router**: 최신 Next.js 기능 활용
- **Server Components**: 서버 사이드 렌더링으로 성능 최적화
- **반응형 디자인**: 모바일 우선 설계
- **Tailwind CSS**: 유틸리티 퍼스트 CSS

### 백엔드
- **Serverless Functions**: Vercel Serverless Functions
- **API Routes**: Next.js Route Handlers
- **실시간**: Supabase Realtime

### 데이터베이스
- **PostgreSQL**: Supabase 호스팅
- **RLS**: Row Level Security로 데이터 격리
- **트리거**: 자동 데이터 채움 및 검증
