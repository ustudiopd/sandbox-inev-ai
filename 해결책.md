좋아. “전문적인 보고서” 느낌이 나고, **의사결정에 실제로 도움이 되려면** 보고서가 단순 요약(“34%가…” “28%가…”)을 넘어서 **결정해야 할 것(Decision) → 선택지(Options) → 추천안(Recommendation) → 실행계획(Action) → 리스크/신뢰도(Confidence)**까지 내려와야 해요.

아래는 EventLive에 넣기 좋은 형태로, **보고서 설계(내용 구조) + 데이터 산출(서버 계산) + AI가 해야 할 일(서술/플레이북) + 품질 게이트(Linter) + UI/PDF 구성**까지 한 번에 정리한 “Decision-grade AI Report 설계안”입니다.

# 1) 보고서가 의사결정에 도움 되려면 반드시 있어야 하는 5가지

## 1) Decision Questions가 앞에 있어야 함

보고서 첫 페이지에서 아래 같은 질문에 답을 줘야 “결정에 도움”이 됩니다.

* 이번 캠페인은 **지금 바로 컨택해야 하는 리드가 몇 명**인가?
* 영업 리소스가 제한될 때, **어느 채널(온라인/방문/전화)에 몇 슬롯**을 배정해야 하나?
* 마케팅은 **어떤 메시지/오퍼**로 **어떤 세그먼트**를 먼저 치면 되나?
* 데이터가 작은데, **어떤 결론은 확정/어떤 건 가설**인가?

## 2) 추천안은 “선택지 대비”로 보여야 함

단일 추천만 쓰면 “그럴듯한 주장”이 되고, 선택지 비교를 하면 “전문 보고서”가 됩니다.

예:

* 옵션 A: 온라인 미팅 우선(리소스 효율)
* 옵션 B: 방문 우선(전환율 기대)
* 옵션 C: 혼합(핵심 세그먼트만 방문)

그리고 **각 옵션의 기대효과/리스크**를 붙이면 의사결정이 쉬워져요.

## 3) 모든 주장에 Evidence ID가 붙어야 함

전문 보고서는 “말”이 아니라 “근거”로 설득합니다.

* E1: Q1 분포 (N=50, 1주 이내 34%)
* E2: Q1×Q3 교차표 (1주 이내 그룹의 온라인 미팅 비중 35.3% …)
* E3: ‘방문 요청’ 수량(14/50)
* E4: 셀 표본 N<5 경고

이렇게 **근거 카탈로그(Evidence Catalog)**를 먼저 만들고, 보고서 문장/결정카드가 E1/E2를 참조하게 만들면 “제대로 된 분석 느낌”이 확 올라갑니다.

## 4) Action은 “누가/언제/몇 명/목표 KPI”가 있어야 함

“강화해야 합니다”는 금지. 대신:

* Owner: Sales / Marketing / Ops
* Due: 24h / 7d / 14d
* Target: 17명에게 1차 컨택, 미팅 8건 확정
* KPI: 미팅전환율, 노쇼율, PoC 신청율…

## 5) Confidence(확정/가설) 레벨을 명시해야 함

특히 N=50이면, 교차 셀 표본이 작을 수 있어요.
그래서 보고서는 결론마다 반드시 아래를 붙여야 합니다:

* **Confirmed**: 표본 충분 + 단순 분포 기반
* **Directional**: 교차표 기반이나 일부 셀 N 작음
* **Hypothesis**: N 부족, 추가 질문 필요

# 2) “Decision-grade 보고서” 권장 목차 (UI/PDF 모두 동일)

## 0. 신뢰성 프리앰블 (고정 문구)

너가 원했던 그 문구 그대로 맨 앞에 고정 삽입(서버 템플릿).

## 1. 분석 대상 요약 (1분 요약)

* 캠페인명 / 분석시점 / 응답수(N) / 문항수 / 렌즈(sales/marketing/general)

## 2. References 요약 (AAPOR/ISO/ESOMAR + BANT/MEDDIC)

* “우리는 어떤 기준으로 해석했는지”를 5줄로.

## 3. KPI Snapshot (도넛/막대) — **“현황을 30초에 끝내는 구간”**

* 핵심 문항 3~6개의 도넛
* TOP 선택지 + (분자/분모) 표시
* 여기까지는 “현황”.

## 4. Decision Cards (가장 중요, 3~5장)

각 카드 구성 고정:

* **Decision Question**
* Options (A/B/C)
* **Recommendation** (우리가 추천하는 옵션)
* Evidence (E1,E2…)
* Expected Impact (KPI 변화 가정)
* Risks / Mitigation
* Confidence (Confirmed/Directional/Hypothesis)

> 이 섹션이 들어가야 “전문 보고서” 느낌이 납니다.

## 5. Action Board (24h / 7d / 14d)

* 업무지시 보드 형태(체크리스트)
* 대상 수량/담당/마감/KPI 포함

## 6. Playbooks

* 세그먼트별 토크트랙(콜 오프너/질문/CTA/반박)
* 마케팅팩(제목/CTA/랜딩훅/소재/채널) + “누구에게 보내는지” 명확히

## 7. Data Quality & Limits

* “어디까지 확정이고, 어디부터 가설인지”를 다시 정리
* 후속 설문 개선 추천(추가 문항)

## 8. Appendix

* 교차표/리드 큐 요약표/세그먼트 정의표
* (선택) CSV 다운로드 링크/설명

# 3) 서버가 “결정 지원용 근거”를 만들어야 AI가 전문적으로 보인다

AI가 계산하면 흔들립니다. 그래서 **근거는 서버가 만들고** AI는 글/액션만 쓰게 해야 해요.

## 서버가 반드시 만들어야 하는 Deterministic 산출물

### A) Evidence Catalog 생성

* 각 수치의 원천을 ID로 관리 (E1~E20)
* 예:

  * E1: Q1 분포
  * E2: Q3 분포
  * E3: Q1×Q3 crosstab + rowPct + overallPct + lift
  * E4: 표본/결측/셀 N<5 경고

### B) Crosstab Highlights (상위 5개 “의미 있는 차이”만)

* AI에게 원본 매트릭스를 다 주지 말고,
* “의미 있는 변화”만 추려서 주면 AI가 빈말을 못합니다.

예 포맷:

* `highlight`: "1주 이내 그룹의 온라인미팅 비중 35.3% vs 전체 26.0% (lift 1.36, 6/17)"
* `confidence`: Directional (셀N>=5면 Confirmed)

### C) Capacity/Resource Plan (의사결정용)

영업/마케팅은 “할 수 있냐”가 핵심이라,

* P0 대상 수
* 필요한 미팅 슬롯 수
* 필요한 SE 동행 슬롯 수(방문/PoC)
  를 서버가 계산해서 보고서에 넣어야 함.

---

# 4) AI에게 요구할 출력도 “전문 보고서형”으로 바꾸자

지금처럼 Executive Summary + Marketing Pack만 뽑으면 깊이가 얕아 보입니다.
AI는 아래 2가지만 만들게 하면 됩니다.

## (1) Decision Cards JSON

* 질문/옵션/추천/근거ID/리스크/신뢰도 포함

## (2) Playbooks JSON

* 세그먼트별 토크트랙/메일/랜딩훅/CTA

나머지(숫자, 그래프)는 서버가 렌더링.

---

# 5) 품질을 “자동으로” 올리는 Linter 규칙 (이게 없으면 다시 허접해짐)

보고서가 전문적이려면, 생성 후 저장 전에 **무조건 아래를 검사**하세요.

## Fail 조건(저장 금지, 재시도)

* Decision card가 3개 미만
* Recommendation에 Evidence ID가 2개 미만
* Action Board에 “대상 수량”이 한 개도 없음
* “중요합니다/필요합니다/강화해야” 같은 문장이 action 없이 등장
* Data Quality가 빈 문장/플레이스홀더 포함

## Pass 조건(전문가 느낌)

* 모든 Recommendation이 “Option 대비”를 포함(A/B/C 중 선택)
* 모든 Action이 “Owner/Due/KPI”를 포함
* Confidence가 카드마다 명시됨

# 6) UI/PDF 구성 (너가 원한 순서 유지 + 의사결정 섹션 추가)

너가 원한 “상단 요약 흐름”은 그대로 유지하되, 그 다음에 **Decision Cards → Action Board**를 넣으면 됩니다.

### 화면/ PDF 순서

1. 고정 신뢰 문구
2. 분석대상 요약
3. 레퍼런스 요약
4. 도넛 요약(핵심 문항)
5. **Decision Cards (3~5)**
6. **Action Board (24h/7d/14d)**
7. Playbooks(영업/마케팅)
8. Data Quality & Limits
9. Appendix(교차표/표)

# 7) Cursor에 던질 구현 명세 (핵심만, 복잡도 과하지 않게)

아래를 그대로 Cursor에 넣으면 “전문 보고서/의사결정형” 구조로 바뀝니다.

```md
# [Cursor Spec] Decision-grade AI Survey Report v3

## 목표
- 보고서가 ‘현황 요약’이 아니라 ‘의사결정 지원 문서’가 되도록 개선
- 상단(신뢰문구→대상→레퍼런스→도넛) 이후에 Decision Cards + Action Board를 필수로 추가
- 숫자/그래프 근거는 서버에서 deterministic 생성, AI는 의사결정/액션 문장 생성

## 서버 산출(필수)
1) evidenceCatalog[] 생성 (E1..En)
- item: { id:"E1", title, metric, valueText, n, source:"qStats|crosstab|derived", notes? }

2) crosstabHighlights[] 생성 (최대 5개)
- item: { id:"H1", description, evidenceIds:["E3"], confidence:"Confirmed|Directional|Hypothesis" }

3) capacityPlan 생성
- { p0Count, meetingSlotsNeeded, visitSlotsNeeded, seSlotsNeeded, suggestedSLA }

## AI 출력(스키마 단순)
- aiDecisionPack:
  - decisionCards[] (3~5)
  - actionBoard: { d0[], d7[], d14[] }
  - playbooks: salesScripts[], marketingAssets[]

## Linter(저장 전)
FAIL if:
- decisionCards < 3
- any decisionCard missing evidenceIds
- actionBoard에 targetCount/kpi 없는 항목만 존재
- placeholder 텍스트 포함

## UI/PDF
섹션 순서:
1) trust statement(고정)
2) analysis target
3) references summary
4) donut charts
5) decision cards
6) action board (24h/7d/14d)
7) playbooks
8) data quality/limits
9) appendix (crosstabs)

## PDF
- print view에서 위 순서를 그대로 렌더
- donut charts는 snapshot 기반으로 고정
```

# 8) 마지막으로: 지금 수준에서 “전문가 느낌”을 확 올리는 3가지 추가 요소

데이터가 3문항뿐이어도 아래가 들어가면 보고서가 갑자기 ‘컨설팅 문서’처럼 보입니다.

1. **Decision Card에 옵션 A/B/C 비교표**(간단히)
2. **Action Board에 KPI 목표치**(미팅 n건, PoC n건 등)
3. **Confidence 라벨**(Confirmed/Directional/Hypothesis)

---

## 결론

지금 너희 보고서는 “좋아지고 있는 중”인데, **전문 보고서/의사결정 지원 문서**가 되려면
**Decision Cards + Action Board + Evidence ID + Confidence** 이 4가지를 구조적으로 고정해야 합니다.
